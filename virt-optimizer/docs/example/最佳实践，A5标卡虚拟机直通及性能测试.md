# 1. 概述

## 1.1 文档目的

本文用于指导在昇腾 Atlas 350 服务器上，基于 Ascend 950 NPU 卡完成 NPU 直通虚拟机的创建，并开展以下验证工作：

* 创建 1/2/4/8 卡 NPU 直通虚拟机；
* 在多卡直通虚拟机中进行 UB 互联带宽测试和 P2P 通信测试；
* 在 NPU 直通虚拟机中部署 DeepSeek R1 和 Qwen3 32B，并进行推理性能测试；
* 验证同 batch 场景下，虚拟机推理性能相比裸金属平均达到 90% 以上。

## 1.2 适用范围

| 类型    | 要求                       |
| ----- | ------------------------ |
| 硬件平台  | Atlas 350 服务器            |
| NPU 卡 | Ascend 950 NPU，数量 8 张及以上 |
| 操作系统  | openEuler 24.03 LTS SP3  |
| 服务器架构 | x86                      |

## 1.3 验证目标

1. 验证 1/2/4/8 卡 NPU 可直通到虚拟机，并支持一台主机基于 1/2/4/8 卡粒度拉起多个虚拟机。
2. 验证 Ascend 950 NPU 直通到虚拟机后，虚拟机内支持 NPU 卡 UB 互联和 P2P 互联，且 UB 互联带宽相比物理机性能下降小于 10%。
3. 验证 NPU 直通虚拟机中可部署 DeepSeek R1 和 Qwen 32B，且同 batch 场景下，虚拟机推理性能相比裸金属平均达到 90% 以上。

# 6. 虚拟机创建和 NPU 设备直通

> **说明：**
>
> 1. 执行本章节脚本前，请先完成第 3 章中的昇腾软件部署。
> 2. NPU 直通虚拟机后，如需在虚拟机内测试 P2P 通信或 UB 互联带宽，请登录虚拟机，并在虚拟机内完成第 3 章对应的昇腾软件部署。

## 6.1 创建普通虚拟机

执行以下命令创建普通虚拟机：

```bash
bash create_iso_ks_vm.sh --vm {vm_name}
```

参数说明：

```text
{vm_name}：用户自定义的虚拟机名称。
```

脚本执行过程中会提示输入虚拟机 root 密码：

```text
请输入 VM root 密码:
请再次输入 VM root 密码:
```

脚本执行完成后，虚拟机 OS 重启需要一段时间。可通过以下命令进入串口，查看虚拟机 OS 启动状态：

```bash
virsh console {vm_name}
```

普通虚拟机创建完成后，可执行以下命令查询虚拟机网络地址 `Address`，并登录虚拟机：

```bash
virsh domifaddr {vm_name} --source lease
ssh root@{Address}
```

如需从宿主机向虚拟机传输文件，可使用 `scp` 命令。

示例：将宿主机文件拷贝到虚拟机 `/root` 目录：

```bash
scp -r {宿主机文件} root@{虚拟机IP}:/root
```

如需删除虚拟机，请提前确认虚拟机中的软件和文件已备份，或确认不再需要。

## 6.2 给虚拟机直通 NPU


## 6.3 给虚拟机移除 NPU

如果需要移除虚拟机中的所有 NPU，以便调整虚拟机直通的 NPU 数量，执行以下命令：

```bash

```

脚本执行完成后，可在宿主机执行以下命令确认 NPU 是否已释放：

```bash

```

> **说明：**
> NPU 卡释放并重新上线到宿主机可能存在一定延迟。如果未及时显示，请等待一段时间后再次查看。

确认虚拟机中的 NPU 已移除，且 NPU 已重新上线到宿主机后，可再次执行第 6.2 节中的直通命令，重新为虚拟机直通指定数量的 NPU。

# 7. P2P 通信测试

## 7.1 物理机 P2P 带宽测试

> **说明：**
> 如需对比物理机和虚拟机的 P2P 带宽，请先完成物理机和虚拟机的昇腾软件部署。

执行以下命令，在物理机上进行 NPU 卡之间的 P2P 带宽测试：

```bash
ascend-dmi --bw -t p2p --dd {npu卡1} --ds {npu卡2} --et 100 -s 8 -q
```

参数说明：

```text
--dd {npu卡1}：表示通信的目标设备为 NPU 卡 1。
--ds {npu卡2}：表示通信的源设备为 NPU 卡 2。
```

示例：

```bash
ascend-dmi --bw -t p2p --dd 0 --ds 2 --et 100 -s 8 -q
```

## 7.2 虚拟机 P2P 带宽测试

> **说明：**
> 测试前，请先完成虚拟机内的昇腾软件部署。

登录虚拟机后，执行以下命令进行虚拟机内 NPU 卡之间的 P2P 带宽测试：

```bash
ascend-dmi --bw -t p2p --dd {npu卡1} --ds {npu卡2} --et 100 -s 8 -q
```

参数说明：

```text
--dd {npu卡1}：表示通信的目标设备为 NPU 卡 1。
--ds {npu卡2}：表示通信的源设备为 NPU 卡 2。
```

示例：

```bash
ascend-dmi --bw -t p2p --dd 1 --ds 3 --et 100 -s 8 -q
```

示例输出如下：


## 7.4 性能差异说明

一般情况下，UB 互联场景下的 NPU 卡在虚拟机和物理机中的带宽性能差异较小，性能差异通常小于 3%。
